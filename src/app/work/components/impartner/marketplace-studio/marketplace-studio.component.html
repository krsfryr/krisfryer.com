Defining the Challenge

From Manual Builds to Scalable Simplicity

Impartner's original partner marketplace solution was never designed for scale. Every client required a custom build from our internal team resulting in dozens of fragmented, one-off codebases that were difficult to maintain, update, or standardize.

It was clear we needed to reimagine the experience as a scalable, embeddable, and self-service tool that partners could launch without relying on engineering.

Painful Manual Process

  • Developers manually built every single marketplace instance
  • No UI or tools existed for clients to manage it on their own
  • Updates required engineering time for even minor changes
  • The lack of standardization created inconsistent partner experiences

“It took weeks to launch something simple and every request pulled a dev off more strategic work.”

A Universal, Flexible Solution

  • Design a customizable UI that just works in any partner's website
  • Empower partners to self-manage marketplaces without code
  • Create consistency in layout, branding, and UX across clients
  • Build a scalable foundation that could support hundreds of implementations with minimal dev overhead

"What if we could create a fully customizable marketplace that just works out of the box?”

Defining the Solution

From Custom Chaos to Scalable Simplicity

How we turned feedback into a flexible, white-labeled Marketplace system.

With Impartner's strong preference for high-fidelity output and an existing design system in place, I jumped straight into Figma. Past client implementations had already revealed what worked and what didn't, so I used those real-world learnings as my foundation. From the start, we prioritized flexibility, reusability, and brandability; key pillars for a scalable SaaS feature that would need to serve dozens of unique clients.

Instead of asking users to configure confusing object relationships, we built a solution that worked behind the scenes. Two dedicated applications, one for Partner Marketplaces and one for Solutions, which gave clarity to users while preserving backend logic.

We weren't just designing for the average case.
We were designing for edge cases at scale.

Giving Clients the Power to Customize Without Overwhelm

From early versions, clients could:

  • Choose between list or card layouts
  • Set the number of listings displayed
  • Customize branding colors for key elements like buttons and links
  • Define partner listing content and header configurations

We deliberately limited choices to avoid complexity, striking a balance between flexibility and usability.

Systematizing the UI

Even with an existing design system, this project required a new set of components.

I designed custom filters, search layouts, listing cards, and configuration tools inside Marketplace Studio, all while ensuring visual and functional consistency across the broader platform.

We designed a lightweight theming system that gave clients control without compromising accessibility or brand cohesion. The branding tab allowed customers to select primary colors that seamlessly styled CTAs and links without opening the door to design chaos.

Iteration & Feedback Loops

Designs were tested in real environments, not just theory.

I ran user tests using interactive prototypes and collaborated directly with Early Access (EA) clients as they explored in-progress builds. These feedback loops revealed edge cases, surfaced feature requests, and shaped the UI into something clients actually wanted to use. These weren't abstract UX decisions, they were sharpened by client reality.

Internally, I collaborated with the team in bi-weekly review sessions to tackle blockers and continuously improve the experience.

Cross-Functional Delivery

Design handoff wasn't just a phase, it was baked into the process.

After finalizing a feature in Figma, I marked it as “dev ready,” triggering assignment by the PM for the upcoming sprint. Developers always had clarity on what was coming next, minimizing churn and speeding up implementation.

Collaborating with Product, Engineering, and Onboarding from the beginning meant that everyone had shared context and we could move fast without breaking things.

Designed for scale. Proven in the wild.
Not just theory, this was shaped by real clients, real data, and real stakes.

Collaboration & Handoff

Seamless Collaboration & Dev-Ready Handoff

We met with engineering twice a week to align on progress, clarify edge cases, and adapt to technical constraints. All designs were created in Figma, with responsive breakpoints and clearly defined states for buttons, filters, and tabs.

Developers used Figma's Dev Mode for handoff, supported by annotations and comments that called out behavior, logic, and hierarchy. We prioritized clarity and consistency to avoid back-and-forth, and our tight collaboration helped us ship on schedule.
