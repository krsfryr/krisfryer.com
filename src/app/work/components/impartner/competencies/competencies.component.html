Solution to Showcase Partner Achievements in PRM Platforms

In today's partner ecosystems, showcasing the strengths and expertise of partners is crucial for driving business success. However, many Partner Relationship Management (PRM) platforms lack the necessary tools to effectively represent partner achievements at the account level. As a UX/UI designer at Impartner, I was tasked with designing a Competency Management solution that would allow administrators to define and display account-level competencies, such as certifications, specialties, and awards, while seamlessly integrating these features into the broader PRM ecosystem.

This project not only aimed to address the gaps in existing tools but also to provide clients with a streamlined, flexible way to manage and present partner competencies, enabling better segmentation, improved partner discoverability, and more robust reporting. The result is a user-friendly interface that empowers PRM admins to enhance their partners' visibility and success within the marketplace.

Software

Challenges

Incomplete Competency Representation

Current PRM tools only support user-level achievements like Journeys and Certifications, falling short in representing account-level competencies such as certifications, awards, and accreditations. This gap limits clients' ability to showcase partner expertise.

Limited Customization & Flexibility

Existing solutions lack the flexibility to segment partners based on multiple competencies or integrate those competencies into features like Partner Marketplace, PCM tiers, and business planning. This limits admins' ability to create a personalized partner experience.

Problem Statement

How might we enable PRM administrators to effectively manage and display account-level competencies in a flexible and customizable way that drives business success?

User Flows & Wireframes

We developed a robust Competency Management system within the PRM platform that allows administrators to manually define and assign multiple competencies to partner accounts. Currently, this system only integrates with Partner Marketplace, but will eventually integrate seamlessly across different features, such as Rewards, Business Plans, and Training & Certifications. The solution includes:

  • Add/Edit Page: A user-friendly interface for administrators to manage competencies, including logo, title, type, description, and assignment details (e.g., assigned date, expiration date).
  • Marketplace Display Integration: Competency information such as name and image can be used as filters or displayed on listings within the Partner Marketplace.
  • Partner Portal Competency Page: Displays earned and available competencies, showing relevant details such as image, title, description, and dates.

Hi-Fi Design

Create New Competency

This is the "Create New Competency" screen from the Competency Management application. This initial prototype was designed primarily for desktop devices, as the platform is predominantly used in desktop environments. Given Impartner's business model, mobile optimization is less of a priority for applications within the PRM ecosystem. This allows the design to focus on delivering the best experience for desktop users.

Heading

One key requirement was ensuring users could easily identify and update the published state of the competency. Initially, we explored various design patterns, including toggles and dropdowns, to manage this status. However, since autosave is not a feature in the PRM application, using a toggle didn't align well with the existing pattern.

After considering placement options, we decided that placing the status control in the header would provide the best user experience. However, a dropdown in the header disrupted the flow. Ultimately, we landed on a segmented control, which proved to be the ideal solution. It not only clearly displays the current status (Published or Unpublished) but also allows users to switch between states and easily save their changes.

Properties

For the Competency Properties section, we needed to gather key details like the name, type, partner description (with a condensed WYSIWYG editor), public description, and an image upload for the badge. Most fields were required, except for the public description, which we made clear to the user.

We decided on a two-column layout: all the text fields are on the left, while the image upload sits on the right. This approach not only follows best design practices but also helps users focus on filling out the content and uploading an image separately. The two-column design also keeps the page visually balanced, making it easier to navigate without feeling overwhelming.

Accounts

In the Accounts section, users can easily add and manage the accounts associated with a specific competency. To keep things efficient, we've added the ability to search through the added accounts, making it simple to find what you need quickly.

For each account, you can specify an Assigned Date and set an optional Expiration Date, which can be left blank if you want the competency to never expire. To simplify management, users can edit these dates individually or use the bulk edit feature to update the Assigned and Expiration Dates for multiple accounts at once.

If you need to remove an account, you can either do it one at a time or use the bulk remove option for a more streamlined process when managing several accounts at once. These features make it easier to maintain control over which accounts are assigned to a competency and when those assignments will expire.

Competencies Page (Partner Facing)

The Competencies page was designed to provide a clear and concise overview of a partner's achievements and qualifications. By showcasing both "Current" and "Available" badges in a single view, we aimed to improve user experience and reduce the need for additional clicks.

To ensure clear differentiation between earned and available badges, we employed a combination of color and visual hierarchy. This design approach allows users to quickly identify their current status and explore potential opportunities for advancement.

By creating a visually appealing and informative Competencies page, we have successfully enhanced the partner experience and provided a valuable resource for understanding and tracking progress within our program.

Current

This section showcases the badges and achievements you've earned. To make the information visually appealing and easy to understand, we designed a layout that prioritizes clarity and organization.

The badge name, the most important detail, is displayed in a larger font size to immediately grab attention. Each badge also has a category, which is positioned above the name to provide context. The earned or assigned date is placed below the title for easy reference.

To ensure a clean and consistent presentation, we aligned the badge images to the right and the accompanying text to the left. This layout makes it simple to scan the information and quickly identify the badges and their details.

Available

The available section lists all the badges that partners can earn. To visually distinguish it from the current section, we used a colored background and placed the badge images within a container to prevent any color conflicts.

To make it easy for partners to claim completed badges, we added an 'Add claim' button. Clicking this button opens a modal where the user must provide a description of their accomplishments and upload any supporting documentation. Once submitted, the badge will display a 'Pending approval' status. Once submitted, the user can also edit their claim if needed.

Key takeaways

Balancing Functionality and Simplicity

One of the biggest challenges in this project was ensuring the interface had all the necessary features without overwhelming the user. For instance, instead of using more complex options like dropdowns or toggles, we opted for a segmented control in the header to manage the competency status. This made it much easier for users to find and update key information while keeping the interface clean and intuitive.

Flexibility for User Needs

Another key takeaway was making sure the system was flexible enough to meet different user needs. We understood that users might need varying levels of control, so we added options like leaving expiration dates blank for competencies that never expire, and bulk editing tools to save time when managing multiple accounts. This flexibility was crucial in making the system both efficient and user-friendly.
